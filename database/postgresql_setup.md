# PostgreSQL Database Setup Guide

This guide outlines how to configure a PostgreSQL database for the Personal Portfolio Website.

---

## 1. Local PostgreSQL Setup (Windows/macOS/Linux)

### Installation
- **Windows**: Download and run the installer from [EnterpriseDB](https://www.enterprisedb.com/downloads/postgres-postgresql-downloads).
- **macOS**: Install via Homebrew: `brew install postgresql@14`
- **Linux (Ubuntu/Debian)**: Run `sudo apt update && sudo apt install postgresql postgresql-contrib`

### Creating Database and User
Log in to the PostgreSQL prompt:
```bash
psql -U postgres
```
*(If prompted for a password, enter the password configured during installation)*

Execute the following commands to create the database and user:
```sql
-- Create the database
CREATE DATABASE portfolio_db;

-- Create the admin user
CREATE USER portfolio_admin WITH PASSWORD 'SecurePassword123';

-- Grant roles and privileges
ALTER ROLE portfolio_admin SET client_encoding TO 'utf8';
ALTER ROLE portfolio_admin SET default_transaction_isolation TO 'read committed';
ALTER ROLE portfolio_admin SET timezone TO 'UTC';
GRANT ALL PRIVILEGES ON DATABASE portfolio_db TO portfolio_admin;
```

Exit the shell:
```sql
\q
```

---

## 2. Environment Variables Configuration
To link Django with your PostgreSQL instance, set the following environment variables (e.g., in your local `.env` or Render settings):

```env
DATABASE_URL=postgres://portfolio_admin:SecurePassword123@localhost:5432/portfolio_db
SECRET_KEY=your_production_secret_key_here
DEBUG=False
ALLOWED_HOSTS=yourdomain.com,localhost,127.0.0.1
```

If these environment variables are not found, the portfolio website will automatically fall back to a local `db.sqlite3` file inside the `backend/` folder to ensure the application remains functional immediately.

## 3. SQL Tables Reference (Generated by Django migrations)

Here is the raw PostgreSQL schema for reference:

### Projects Table (`api_project`)
```sql
CREATE TABLE api_project (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title varchar(200) NOT NULL,
    description text NOT NULL,
    image varchar(100), -- Holds file path to uploaded files
    image_url varchar(200), -- Holds direct fallback link
    tech_stack varchar(500) NOT NULL,
    github_link varchar(200),
    live_link varchar(200),
    created_at timestamp WITH TIME ZONE NOT NULL
);

-- Index for ordering
CREATE INDEX api_project_created_at_idx ON api_project (created_at DESC);
```

### Certificates Table (`api_certificate`)
```sql
CREATE TABLE api_certificate (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title varchar(200) NOT NULL,
    issuing_organization varchar(200) NOT NULL,
    issue_date date NOT NULL,
    credential_id varchar(200),
    credential_url varchar(200),
    file varchar(100), -- File path for uploaded certificate PDF/Image
    file_url varchar(200),
    created_at timestamp WITH TIME ZONE NOT NULL
);

-- Index for ordering by issue date
CREATE INDEX api_certificate_issue_date_idx ON api_certificate (issue_date DESC);
```

### Contact Messages Table (`api_contactmessage`)
```sql
CREATE TABLE api_contactmessage (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name varchar(150) NOT NULL,
    email varchar(254) NOT NULL,
    subject varchar(200),
    message text NOT NULL,
    created_at timestamp WITH TIME ZONE NOT NULL,
    read_status boolean NOT NULL DEFAULT FALSE
);

-- Index for listing messages by date
CREATE INDEX api_contactmessage_created_at_idx ON api_contactmessage (created_at DESC);
```
